It is the de facto orchestrator contract.
- [src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L39) owns CLI backend execution: workspace fallback, prompt construction, session resume/restart rules, image handling, MCP config injection for Claude CLI, stdout parsing, process cleanup, and usage/session-id extraction from CLI output.

- [src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L98) owns direct Gemini API calls plus its own persisted conversation history file, prompt construction, streaming SSE parsing, native tool execution loop, and cumulative token accounting.
- [src/agents/cli-runner/helpers.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner/helpers.ts#L255) is the shared prompt/runtime adapter in practice: it builds the agent system prompt, normalizes CLI models, parses CLI outputs, and serializes CLI execution.

- [src/agents/agent-scope.ts](/Users/grimreaper/.hell/anima/src/agents/agent-scope.ts#L74) is the shared session-agent identity resolver used to decide agent scope and heartbeat eligibility.
- [src/agents/system-prompt.ts](/Users/grimreaper/.hell/anima/src/agents/system-prompt.ts#L265) is the real prompt contract: it conditions behavior on tool availability, memory tools, messaging, mission mode, docs, and runtime identity.
- [src/agents/cli-session.ts](/Users/grimreaper/.hell/anima/src/agents/cli-session.ts#L4) is the only normalized persistent session-id helper, but it currently applies only to CLI-backed sessions.

## 2) Contract drift list

- Tool contract drift: CLI passes `tools: []` into prompt construction and relies on native CLI capabilities instead ([src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L119)); Anthropic direct also advertises no tools ([src/agents/anthropic-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/anthropic-direct-runner.ts#L153)); Gemini direct exposes full ANIMA tools and executes them inline ([src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L131)). Same runner contract, different capability model.
- Session continuity drift: CLI resumes provider-native sessions via `cliSessionId` and transcript metadata ([src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L133)); Anthropic and Gemini each persist separate ad hoc history sidecars (`.anima-history.json`, `.gemini-history.json`) ([src/agents/anthropic-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/anthropic-direct-runner.ts#L47), [src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L48)). There is no unified session-state interface.
- Streaming contract drift: top-level `runNoxSoftEmbeddedAgent` assumes partial text chunks and emits lifecycle events itself ([src/agents/noxsoft-runner.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.ts#L314)); CLI streams normalized stdout snapshots ([src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L303)); Anthropic direct emits one full text after completion ([src/agents/anthropic-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/anthropic-direct-runner.ts#L258)); Gemini streams cumulative full-text snapshots on every chunk ([src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L303)). Consumers cannot rely on chunk semantics.
- Assistant-start drift: `noxsoft-runner` tries to centralize assistant-start in `emitPartial` ([src/agents/noxsoft-runner.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.ts#L314)), but both direct runners also call `onAssistantMessageStart` themselves ([src/agents/anthropic-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/anthropic-direct-runner.ts#L258), [src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L278)). This is double-ownership of the same lifecycle boundary.
- Result-shape drift: the canonical type includes `systemPromptReport`, `promptTokens`, `lastCallUsage`, messaging outputs, and `sessionId` ([src/agents/noxsoft-runner.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.ts#L37)), but the direct runners populate only a subset, and Gemini never returns `timeout` despite detecting timeout conditions ([src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L381)).
- Error taxonomy drift: CLI maps failures into `FailoverError` with classified status ([src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L344)); direct runners return plain `status/meta.error.kind` objects. The orchestrator’s auth-profile rotation depends on inferred error kinds/messages rather than a stable error contract ([src/agents/noxsoft-runner.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.ts#L146)).
- Model normalization drift: `noxsoft-runner` normalizes requested refs at the top ([src/agents/noxsoft-runner.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.ts#L314)); CLI uses backend-specific aliases ([src/agents/cli-runner/helpers.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner/helpers.ts#L309)); Anthropic and Gemini each carry local `MODEL_MAP`s. There is no single provider/model canonicalization path.
- Prompt policy drift: all runners reuse the same prompt builder, but they mutate `extraSystemPrompt` differently to compensate for backend differences ([src/agents/cli-runner.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner.ts#L90), [src/agents/anthropic-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/anthropic-direct-runner.ts#L153), [src/agents/gemini-direct-runner.ts](/Users/grimreaper/.hell/anima/src/agents/gemini-direct-runner.ts#L174)). Capability policy is being encoded as prompt text instead of contract data.
- Test coverage drift: current tests verify strategy selection and profile fallback, not cross-runner behavioral equivalence ([src/agents/noxsoft-runner.test.ts](/Users/grimreaper/.hell/anima/src/agents/noxsoft-runner.test.ts#L74)).

## 3) Recommended shared abstractions

- `RunnerCapabilities`: explicit per-strategy declaration for `tools`, `images`, `nativeSessionResume`, `streamingMode`, `toolCallingMode`, `reasoningControl`, `promptToolAdvertisement`. Stop inferring capability from runner implementation.
- `RunnerSessionStore`: common interface for `loadState`, `appendTurn`, `commitAssistantTurn`, `readResumeHandle`, `writeResumeHandle`. CLI session ids and direct sidecar histories should be different backends behind one contract.
- `PreparedRunContext`: one shared builder that resolves workspace, agent scope, heartbeat, docs path, bootstrap context, tool set, and final system prompt once. Today this setup logic is duplicated across all three concrete runners.
- `RunnerEventSink`: normalize events to `run_started`, `assistant_started`, `delta`, `tool_call_started`, `tool_call_finished`, `run_completed`, `run_failed`, `run_timed_out`. The orchestrator should own emission; leaf runners should emit structured events, not call user callbacks directly.
- `NormalizedRunOutcome`: separate final outcome from transport details. Require stable fields for `status`, `provider`, `model`, `session`, `usage`, `error`, `payloads`, `capabilitiesUsed`, `toolCalls`, `systemPromptDigest/report`.
- `ProviderAdapter`: minimal backend adapter that converts `PreparedRunContext` plus session state into provider-specific wire format and back. Keep API/CLI transport differences here, not spread across runner files.
- `ToolExecutionFacade`: if tools are allowed, execute through one shared pipeline. Gemini’s inline loop should be promoted into a reusable tool-turn engine or disabled until other providers can match the contract.

## 4) Concrete implementation order

1. Define the contract types first in one place: `RunnerCapabilities`, `PreparedRunContext`, `RunnerEvent`, `RunnerSessionStore`, `NormalizedRunOutcome`, and a typed `RunnerFailureKind`.
2. Extract shared run preparation from the three runners into a single `prepareRunnerContext()` that wraps [workspace-run.ts](/Users/grimreaper/.hell/anima/src/agents/workspace-run.ts#L72), [bootstrap-files.ts](/Users/grimreaper/.hell/anima/src/agents/bootstrap-files.ts#L48), [agent-scope.ts](/Users/grimreaper/.hell/anima/src/agents/agent-scope.ts#L74), and [cli-runner/helpers.ts](/Users/grimreaper/.hell/anima/src/agents/cli-runner/helpers.ts#L255).
3. Introduce `RunnerSessionStore` with two adapters: CLI resume-handle storage backed by [cli-session.ts](/Users/grimreaper/.hell/anima/src/agents/cli-session.ts#L4) and direct-history storage backed by the current sidecar files.
4. Move lifecycle and partial emission ownership fully into `noxsoft-runner`; make concrete runners return structured event streams or buffered events, never call `onAssistantMessageStart` directly.
5. Normalize failure mapping across CLI and direct paths so auth/rate-limit/timeout/billing are emitted from one classifier, then keep auth-profile rotation in the orchestrator.
6. Decide tool policy explicitly: either `no tools for all direct runners` for parity, or lift Gemini’s tool loop into a shared provider-agnostic tool-turn engine and adopt it incrementally.
7. Add conformance tests that run the same fixture turn through each strategy and assert identical event ordering, status mapping, session persistence semantics, and usage/error shape.

## 5) Top 5 risks

- Capability skew is already user-visible: Gemini direct can execute ANIMA tools while Anthropic direct and CLI paths cannot. Unifying the contract will force a product decision, not just refactoring.
- Session migration risk is high: CLI resume ids and direct sidecar histories represent different continuity models. A bad unification can silently break multi-turn context.
- Event ownership is currently split between orchestrator and leaf runners. Refactoring without a strict event contract can easily introduce duplicate starts, duplicated partials, or missing final messages.
- Prompt behavior currently compensates for missing capability metadata. Moving that logic without preserving prompt invariants can change agent behavior even if the transport still works.
- Test coverage is insufficient for safe convergence. Without cross-runner contract tests, the refactor will likely preserve type compatibility while regressing semantics.
